# DORA ICT ASSET REGISTER: AI INFRASTRUCTURE
## Version: 1.0 | Review cadence: quarterly and on material change

### REGISTER ENTRY FIELDS

| Field | Details |
| :--- | :--- |
| **Asset ID** | Unique identifier for this register entry |
| **Asset Name** | Name of the ICT asset or third-party service |
| **Asset Type** | Compute / Model / Inference Engine / Storage / Observability / Secrets / Identity / Registry / Other |
| **Provider Name** | Legal name of the providing entity |
| **Provider Domicile** | Country of incorporation |
| **Service Region** | Physical region(s) where the service operates |
| **Criticality** | Critical Function / Important Function / Supporting Function |
| **Substitutability** | High / Medium / Low / Single Source |
| **RTO** | Recovery time objective if this dependency fails |
| **RPO** | Recovery point objective if this dependency fails |
| **Concentration Risk** | Note if this provider also supplies other critical assets; flag if a single provider covers more than one critical function |
| **Data in Scope** | Categories of personal or regulated data processed |
| **Contractual Basis** | DPA / MSA / SLA / License / None |
| **Exit Plan** | Documented substitution procedure within RTO, or "Not documented" with scheduled remediation date |
| **Last Reviewed** | Date of last review |
| **Review Owner** | Name and role of responsible person |
